# Bay Ave Barnacle — Flood Prediction Model v0.8

**Target:** Predict whether 342 Bay Ave (Highlands NJ) will flood in
the next 24 hours, with per-landmark depth estimates at 16 named
landmarks around the property.

**Promotion date:** 2026-06-16 (next-day reaction to 2026-06-15 PM
event — a storm-condition tide that v0.7 under-predicted by 1.5″ at
curb).

**Changes from v0.7** — driven by the 2026-06-15 PM event (4-grate
cross-fit, 18 measurements; v0.7 underestimated water at peak by
1.3″ structurally, 3.5″ operationally because forecast also
undershot SH):

1. **Local enhancement: −0.13 ft → 0.00 ft (constant).** v0.8 takes
   the conservative (over-predict) value from the 4 measured events:
   - 3 regular-tide events (5/18 SH 6.58, 5/31 SH 6.17, 6/14 SH 7.13)
     all had **offshore** peak winds (S/SW) and gave enhancement −0.13
   - 1 storm-condition event (6/15 SH 7.289) had **onshore** peak
     winds (N/NNE) after a late veer and gave enhancement ~0
   Main model uses 0 (matches storm conditions, errs on safer side
   for regular tides by ~1.5″ at curb — within tape precision and
   aligned with user preference for false-positive bias).
2. **Wind-direction adjustment** (NEW; reported as a separate
   "expected actual" line alongside the main prediction). Forecast
   wind at peak in the offshore sector → apply −0.13 ft adjustment,
   reported as a secondary estimate. Calibrated against 6/14 and 6/15.
   See `compute_wind_adjustment()` in `forecast/flood_forecast_daily.py`.
3. **New landmark: `sidewalk_under_walkway_lawn_step` at 4.33 NAVD88**
   (cross-fit from 3 6/15 measurements; the sidewalk surface where it
   meets the lawn-step face on the user's walkway).
4. **SH thresholds for all landmarks shift down by 0.13 ft uniformly**
   — everything activates 1.5″ earlier than v0.7 (SH = elev + 2.82
   instead of elev + 2.95).

v0.7 is preserved at `model/archive/v0.7.md` for reference.

If anything in another document disagrees with v0.8, **v0.8 wins.**

## Core formula

```
# Translate Sandy Hook gauge to local water level at 342 Bay Ave
LOCAL_ENHANCEMENT = 0.00 ft                 # v0.8 conservative value
MLLW_TO_NAVD88     = -2.82 ft

water_at_342_NAVD88 = Sandy_Hook_MLLW + LOCAL_ENHANCEMENT + MLLW_TO_NAVD88

# Rain amplification (saturating, applied when peak rain >= 0.1"/hr)
# Inherited from v0.7 unchanged — fits Oct 30 2025 anchor within 0.4".
if peak_rain_rate_in_hr > 0.1:
    rain_add_ft = (8.0 * tanh(peak_rain_rate_in_hr)) / 12.0
    water_at_342_NAVD88 += rain_add_ft

# Per-landmark depth (inches) — purely geometric from one water level
depth(landmark) = max(0, water_at_342_NAVD88 - landmark.elevation) * 12
```

The shortcut: at any Sandy Hook reading X (MLLW), water at 342 Bay
sits at `(X − 2.82)` ft NAVD88 (without rain). So Sandy Hook
threshold for landmark Y NAVD88 is just `Y + 2.82`.

## Wind-direction adjustment (NEW in v0.8)

Reported as a separate "expected actual" line in the worst-case
detail section of the report. Does NOT replace the main prediction.

```
# Pull wind direction at peak hour from NWS hourly forecast
wind_dir = nws_hourly[closest_period_to_peak].windDirection

OFFSHORE = {"S", "SSW", "SW", "WSW", "SSE"}
ONSHORE  = {"N", "NNE", "NE", "ENE", "E", "NNW"}

if wind_dir in OFFSHORE:
    adjustment_ft = -0.13
    sector = "offshore"  # bay-corner water pushed away from Highlands
elif wind_dir in ONSHORE:
    adjustment_ft = 0.00
    sector = "onshore"   # water piled into bay corner; main applies
else:
    adjustment_ft = 0.00
    sector = "neither"   # cross-shore / unknown; main applies
```

The "expected actual" prediction = main_water + adjustment_ft.

## Three pathways (v0.8 thresholds)

### Pathway A — Direct overtopping

- **Onset** (water tops curb): SH ≥ 6.98 ft MLLW
- **Sidewalk under lawn step**: SH ≥ 7.15 ft
- **Lawn step**: SH ≥ 7.40 ft
- **Porch step**: SH ≥ 7.90 ft
- **Sandy 2012**: SH 13.31 (6-min) / 12.03 (hourly) MLLW

### Pathway B — Storm drain back-flow

`grate_SW` (3.52) emerges first at SH ≥ 6.34. Other grates follow in
order. The Bay Ave upstream grate (low-point ~3.64) emerges at SH ≥
6.46 — this is the actual primary feeder of the user's gutter at
walkway.

### Pathway C — Rainfall amplification (unchanged from v0.7)

`8 · tanh(rate) / 12 ft` water-level rise, applied uniformly. Fits
Oct 30 2025 (1.45 in/hr peak) within 0.4″. Open question: whether
the saturating form holds at modest rates (Dec 19's 0.44 in/hr now
falls into the landmark band under v0.8).

### Cold-weather hypothesis (still advisory only, not applied)

Unchanged from v0.7. See `history/reports/cold_weather_retrospective.md`.

## What might break this model

1. **Wind-rule edge cases**: the offshore/onshore rule is a binary
   compass-sector classifier from 1 event each per sector. Cross-
   shore winds and weak winds (< 8 kt) default to "no adjustment"
   (use main = enhancement 0). The first event we get with a
   cross-shore peak wind will test whether that default is right.
2. **Sustained-storm bay setup**: if a storm runs offshore for
   hours before veering onshore at peak (like 6/15), the bay may
   already be at storm-condition levels when the veer happens. The
   wind-at-peak rule treats this as fully onshore, which matches
   the data we have but is plausibly only-half-the-story. Open.
3. **Rain term**: unchanged from v0.7. Fits Oct 30 well; Dec 19's
   0.44 in/hr is now consistent with the landmark anchor (v0.8
   improvement). Open whether saturating form is right at very
   modest rates.
4. **Three new grates were tape-measured in only 1–2 events each**
   (grate_NW, grate_SW, grate_bay_ave_upstream); cross-grate
   consistency on 6/14 and 6/15 (4-grate cross-fit each, spread 0.4–
   0.6″) confirms elevations within ~0.5″.
5. **`grate_bay_ave_upstream` low-point convention** (3.64 NAVD88)
   — the grate top is uneven over ~1.7″ vertical span. Tape readings
   referenced to a different point on the same grate will read
   differently.
6. **Sea-level rise is accelerating** (~5.45 mm/yr post-1980). Will
   silently shift the enhancement away from 0 over time. Re-calibrate
   every ~5 years.

## What v0.8 does NOT yet do (v0.9+ candidates)

- **Quantitative wind term beyond binary sector + magnitude**:
  speed threshold, sustained-vs-veering distinction, fetch direction
  modeling. Currently the wind adjustment is a single −0.13 ft for
  offshore sectors.
- **Antecedent / preceding-storm influence**: how much does
  yesterday's onshore conditions matter for today's tide?
- **Tape-measured storm event with rain** to settle the Dec 19
  rain-term form question.
